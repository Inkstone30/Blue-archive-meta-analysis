# 블루아카이브 캐릭터 스탯·메타 분석

> 데이터 분석 포트폴리오 | EDA → 통계 검증 → 클러스터링 → 예측 모델

---

## 프로젝트 소개

블루아카이브(Blue Archive)는 넥슨게임즈가 서비스하는 서브컬처 모바일 RPG입니다.  
이 프로젝트는 **"스탯 숫자가 메타를 설명할 수 있는가?"** 라는 질문에서 출발합니다.

SchaleDB의 공개 데이터(194명 학생, 48개 스탯 컬럼)를 바탕으로,  
탐색적 분석에서 시작해 통계 검증, 비지도 학습, 지도 학습까지 분석 깊이를 단계적으로 쌓습니다.

---

## 분석 파이프라인

### Phase 1 — EDA (탐색적 데이터 분석)

| # | 질문 | 방법 |
|---|------|------|
| 1 | 역할(딜러/탱커/힐러/서포터)별 스탯 프로파일은 얼마나 뚜렷하게 구분되는가? | 박스플롯, 바이올린 플롯 |
| 2 | 레어도(★1/★2/★3)에 따라 스탯 분포가 어떻게 달라지는가? | 분포 비교, 누적 분포 |
| 3 | 한정 픽업 학생과 상시 학생 간 스탯 차이가 육안으로 보이는가? | 평균 비교 막대그래프 |
| 4 | 지형 적응도(실내/실외/시가지) 패턴 — 어떤 조합이 가장 많은가? | 히트맵, 빈도표 |

### Phase 2 — 통계 검증

| # | 질문 | 방법 |
|---|------|------|
| 5 | ★3 학생이 ★2보다 스탯이 유의미하게 높은가, 아니면 단순 가챠 등급 차별화인가? | 일원 분산분석(ANOVA) + Tukey HSD |
| 6 | 한정 픽업 학생은 실제로 상시 학생보다 스탯 우위를 갖는가? | 독립 표본 t-검정 |
| 7 | EX 스킬 코스트가 높을수록 공격력도 높은 경향이 있는가? | Spearman 상관, 산점도 |
| 8 | 스탯들 간의 상관 구조는 어떻게 형성되어 있는가? | 상관 행렬 히트맵 |

### Phase 3 — 클러스터링 (비지도 학습)

| # | 질문 | 방법 |
|---|------|------|
| 9  | 스탯 조합으로 자연스럽게 분류되는 "메타 유형"은 몇 가지인가? | K-means (Elbow + Silhouette) |
| 10 | 각 클러스터의 스탯 특성은 무엇인가? ("고화력 근거리형" 등 해석) | 클러스터 프로파일링 |
| 11 | 한정 학생은 특정 클러스터에 집중되는가? | 클러스터별 한정 비율 분석 |
| 12 | 공식 역할 분류와 데이터 기반 클러스터는 얼마나 일치하는가? | 교차표, 시각화 |

### Phase 4 — 예측 모델 (지도 학습)

| # | 질문 | 방법 |
|---|------|------|
| 13 | 스탯만으로 ★3 레어도 여부를 얼마나 정확히 예측할 수 있는가? | Random Forest, Logistic Regression |
| 14 | 고레어도를 결정하는 가장 중요한 스탯 요소는 무엇인가? | Feature Importance, SHAP |
| 15 | 한정 학생 여부를 예측할 수 있는가? (인플레이션 지표 검증) | 이진 분류, 혼동 행렬 |

## 폴더 구조

```
bluearc/
├── data/
│   ├── raw/                  # 수집 원본 — 절대 수정하지 않음
│   └── processed/            # 전처리·피처 엔지니어링 결과
├── notebooks/
│   ├── 01_eda.ipynb          # Phase 1: 탐색적 데이터 분석
│   ├── 02_statistics.ipynb   # Phase 2: 통계 검증
│   ├── 03_clustering.ipynb   # Phase 3: K-means 클러스터링
│   ├── 04_prediction.ipynb   # Phase 4: 분류 모델 + SHAP 해석
│   └── _archived/            # 구 감성분석 노트북 (향후 확장용)
├── src/
│   ├── collectors/           # 데이터 수집 스크립트
│   │   ├── collect_students.py
│   │   └── _archived/        # DC인사이드 수집기 (향후 확장용)
│   └── analysis/             # 재사용 분석 함수
├── app/                      # Streamlit 대시보드 (Phase 4 이후 예정)
├── reports/                  # 최종 시각화·보고서 출력물
└── requirements.txt
```

## 데이터 한계

- **출시일 없음**: SchaleDB에 개별 학생 출시일이 포함되지 않아 시계열 분석 불가  
  → 나무위키·Fandom 위키 크롤링으로 보완 가능 (추후 과제)
- **레이드 실적 없음**: 실제 클리어율·사용률 데이터가 없어 "메타 강함"을 스탯으로만 추론
- **스킬 효과 미반영**: EX 스킬의 질적 차이(광역/단일, 버프 종류 등)는 수치화 미포함
- **194명의 표본 크기**: 머신러닝 모델 성능 해석에 주의 필요 (과적합 위험)
- **스탯 인플레이션**: 후기 출시 학생의 스탯이 전반적으로 높아지는 경향이 있어  
  출시 시기가 다른 학생 간 단순 스탯 비교 시 편향 발생 가능

## 실행 방법

### 1. 환경 설정 (처음 한 번만)

```powershell
python -m venv .venv
.venv\Scripts\activate
pip install -r requirements.txt
```

### 2. 학생 데이터 수집

```powershell
python src/collectors/collect_students.py
# → data/raw/students_raw.csv (194명 × 48컬럼) 생성
```

### 3. 노트북 순서대로 실행

```powershell
jupyter notebook
# 브라우저에서 notebooks/ 폴더 열기
# 01 → 02 → 03 → 04 순으로 실행 권장
```
